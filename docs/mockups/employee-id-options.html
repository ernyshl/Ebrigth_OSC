Employee ID Field — Mockup Comparison

Two design options for how the Employee ID is entered in the Add User form.

Option A — Replace

Pure manual entry

Admin must pick the role code and type all 6 digits every time. No auto-generation.

Add User

8 digits total — first 2 from dropdown, last 6 entered manually.

Result for new user: 33______ → admin types 33080012

Branch info in ID? ❌ Lost (admin chooses freely)

Risk of duplicates? ⚠️ Higher

Option B — Override

Auto-fill, editable

Dropdown + 6 digits auto-fill from role/branch. Admin can override either part for custom IDs.

Add User

Auto-filled from branch + sequence. Edit if you need a custom ID.

Result for new user: 33080012 (auto-filled, admin can change)

Branch info in ID? ✅ Preserved (08 = OD)

Risk of duplicates? ✅ Lower
